# GESTIO DE VERSIONS AL GIT

Regla de versions (SemVer) MAJOR.MINOR.PATCH → 1.2.3
    MAJOR: canvis incompatibles (breaking)
    MINOR: funcionalitat nova compatible
    PATCH: bugfix compatible

Per una app ETL:

    canvis en payload/format, canvis de config incompatibles → MAJOR
    afegir funcionalitat (ex: mail) mantenint compatibilitat → MINOR
    arreglar bugs → PATCH
